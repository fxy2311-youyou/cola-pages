📐 Cola 工作流规则 (Workflow)
#规则内容
1 版本管理 所有文件迭代必须新建文件，不覆盖原文件。命名：文件名-vN.html
2 先确认方向再动手 用户让查资料/做研究前，先说清方向+具体要查什么，等用户确认后再动手
3 交付说明 每次给文件时必须说明：①改了什么 ②怎么用 ③已知限制
4 Obsidian同步 最终版内容同步到 Obsidian/Cola/ 文件夹，放之前跟用户确认
5 Dashboard更新 部署新页面→立刻更新dashboard；本地生成文件→放一起更新或用户要求时更新
6 称呼 聊天时叫用户“sisi”
7 数据脱敏 公司名/人名/电话/银行/地址/seed data全部替换，grep验证0残留
8 新Skill同步 安装新skill后必须加到dashboard的Skills表里

⚡ 快捷操作指南
#操作方法
1 上传新页面 GitHub仓库 → gh-pages分支 → Add file → 输入 文件夹名/index.html → 粘贴内容 → Commit
2 删除页面 进入文件夹 → 点 ... → Delete directory → Commit
3 更新页面 找到对应 index.html → 编辑 → 粘贴新内容 → Commit
4 查看链接 https://fxy2311-youyou.github.io/cola-pages/文件夹名/
5 清除浏览器缓存 F12 → Application → Local Storage → 当前网址 → Clear → 刷新
6 Mac显示隐藏文件 Cmd + Shift + .
