▶ 设计系统是什么？

一套完整的视觉语言和组件规范，确保品牌在所有触点保持一致的像素温暖气质。

像素风有复古游戏的仪式感和手账的精心布置感，温暖而不冷硬，个性而不幼稚。

▶ 可以用在哪些场景？

教程页、Landing Page、App工具型页面、图文卡片——四种场景各有对应的设计指南。

✦ SECTION 08 ✦

质检清单

做完设计后逐条对照。P0必须全过，否则打回修改。P1应过，P2加分。

P0 必须全过
品牌六色使用正确（莓果紫为主色，玫瑰红次强调，绿/薰衣草点缀）
没有使用禁忌清单里的任何元素（冷蓝/glassmorphism/圆角>4px/neon/bounce）
背景使用暖粉底（#FFF5F5 或 #FFF0F5），非纯黑纯白
字体正确加载（5层字体各司其职）
像素边框一致（box-shadow偏移 + 实线border，无圆角混用）
有响应式（至少900px断点，移动端不破版）
截图发社交媒体不会被说"又是AI做的通用模板"
每个section布局形式不同（从layouts.md选取不同模式）
clamp()做fluid sizing（标题/间距都用clamp）
没有使用任何HTML默认样式（所有组件从components.md选用）
P1 应过
至少一个section有视觉惊喜（全宽色彩面板/RPG对话框/像素装饰突破）
字号对比足够极端（大>2.4rem，小<0.75rem）
有Scroll Reveal动效 + stagger延迟
使用了像素装饰（莓果/花朵/藤蔓分隔线）
按钮有像素按下效果（box-shadow归零 + translate位移）
::selection用薰衣草紫高亮
至少3种不同颜色的box-shadow出现
布局模式至少用了3种不同的
P2 加分
hover时卡片"浮起"效果（translate + shadow增大）
自定义像素风滚动条
像素角落装饰（pixel-corner）在重要面板出现
像素进度条或数据可视化元素
RPG对话框用于引用/语录场景
像素art装饰（berry/flower/vine）在section间使用
prefers-reduced-motion尊重
Tab/手风琴组件增加交互性
