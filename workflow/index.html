Workflow Breakdown

用AI做一个产品
从0到上线的全流程

一个普通人怎么用AI对话，从零开始做出一个合同管理系统，然后包装展示、部署上线、串联体验。

—— 4步，不写一行代码 ✦

完整工作流

Step 01

Vibe Coding — 用AI对话搭系统

把现有的Word合同模板和Excel订单表发给AI，用自然语言描述需求："我要一个能选客户、选产品、自动算价、生成PDF的系统"。AI一步步生成代码，你负责试用和反馈。

核心方法：分模块迭代。先做出货合同，再做采购，再做订单记录……每次只解决一个问题，积累成一个完整系统。

Cola / Cursor 自然语言描述需求 → 单HTML文件应用
Step 02

做项目介绍页 — 包装你的作品

系统做好了但没人知道你做了什么。写一个展示页，讲清楚：为什么做、做了啥、怎么做到的、踩了什么坑。让别人3分钟看懂你的能力。

同样用AI生成。给它你的素材（截图、数据、故事），它帮你排版成一个好看的单页网站。

AI生成HTML Case Study结构 → 项目介绍页
Step 03

部署到GitHub Pages — 一键上线

把HTML文件上传到GitHub仓库的gh-pages分支，自动变成一个可访问的网站。不需要买服务器，不需要域名，完全免费。

操作就三步：建文件夹 → 放index.html → 等1分钟。链接直接发给别人就能看。

GitHub Pages 免费托管 → 在线链接
Step 04

用iframe嵌入 — 串联完整体验

把可操作的系统Demo用iframe嵌到介绍页里。别人看完"这是什么"，马上就能上手"试一试"。不需要跳转，一页搞定全部体验。

加一个主页做引导：先展示6个功能卡片，点"进入系统"再看完整应用。层次清晰，不会一上来就懵。

iframe嵌入 Macbook外框样式 → 一体化展示

关键心得

  • 不需要会写代码，AI帮你写。你只负责说清楚"我要什么"和"这不对"
  • 分模块做，一次只搞一个功能。别贪心一口气全丢给AI
  • 每个版本都留着，不要覆盖旧文件。出问题可以秒退
  • 光做出来没用，要包装成别人看得懂的东西。展示页就是你的简历
  • 部署上线是最简单的一步，别被"部署"这个词吓到
  • iframe串联体验是加分项——让人看完介绍马上能动手试
